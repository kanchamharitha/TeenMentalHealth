#--Machine Learning Project Explanation--#
1. Project Title & Objective 
Project Title: Teen Mental Health Prediction System
"My project is called the Teen Mental Health Prediction System. 
It is a machine learning application that predicts whether a 
teenager is at risk of depression using factors like social media
usage, sleep, stress, anxiety, and physical activity. The main goal
is to support early detection so that parents, teachers, and healthcare
professionals can provide timely mental health assistance."
2. Problem Statement & Motivation 
"Teenagers are increasingly facing mental health issues due to factors
like excessive social media use, academic pressure, poor sleep, and low 
physical activity. Identifying students at risk early is difficult because 
symptoms are often not noticed in time. My machine learning project predicts 
depression risk using lifestyle and behavioral data to support early intervention and better decision-making."."
3. Dataset & Technology Stack
Dataset Information
Dataset Source: Kaggle (Teen Mental Health Dataset)
Rows: Approximately 1,000 records (mention the actual number if known)
Columns: 12 input features + 1 target column
Target Variable: depression_label
* Features
* Age
* Gender
* Daily Social Media Hours
* Platform Usage
* Sleep Hours
* Screen Time Before Sleep
* Academic Performance
* Physical Activity
* Social Interaction Level
* Stress Level
* Anxiety Level
* Addiction Level
Technology Stack

4. Data Preprocessing & Exploratory Data Analysis (EDA)
I converted categorical features such as gender and platform usage into numerical
values using Label Encoding. Finally, I performed exploratory data analysis (EDA) 
using histograms, bar charts, box plots, and a correlation heat map to understand 
the relationships between features.
5. Feature Engineering
"After preprocessing, I selected the most relevant features affecting depression prediction. 
Lifestyle-related variables such as stress level, anxiety level, addiction level, sleep hours,
physical activity, and social media usage showed a strong relationship with the target variable."
6. Model Building
"I divided the dataset into training and testing sets using an 80:20 ratio. I trained multiple
machine learning algorithms, including Logistic Regression, Decision Tree, Random Forest, Support
Vector Machine, and K-Nearest Neighbors. After comparing their performance, Random Forest produced 
the best results, so I selected it as the final model."
7. Model Evaluation
Evaluation Metrics
* Accuracy
* Precision
* Recall
* F1-Score
* Confusion Matrix
"I evaluated the model using several performance metrics. Accuracy measures overall correctness, 
Precision measures how many predicted positive cases were actually positive, Recall measures 
how many actual positive cases were correctly identified, and the F1-score balances Precision and Recall.
I also used a Confusion Matrix to visualize correct and incorrect predictions."

8. Model Deployment
"After training the Random Forest model, I saved it using the Pickle library. Then I developed a Flask web 
application where users can enter mental health-related information through an HTML form. The application loads
 the trained model, processes the user input, predicts the depression risk, and displays the result instantly."
9. Challenges Faces
"During development, I encountered several challenges such as handling missing values, encoding categorical 
variables, selecting the best algorithm, and resolving dependency issues with Python libraries like Pandas and NumPy. 
I also improved the Flask user interface to make it more interactive and user-friendly."
10. Results
"The Random Forest model achieved high prediction accuracy and performed better than the other algorithms I tested. 
The web application provides real-time predictions, making it easy for users to assess depression risk based on 
behavioral and lifestyle information."
"The model achieved 97% accuracy."
